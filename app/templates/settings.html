Custom notations

Auto-capture commitments from chat patterns · {{ notations | length }} of {{ max_notations }}

A regex that, when matched in a Slack message you send, logs the message as a commitment.

  • 1–128 characters long.
  • ? is not allowed (collides with normal questions).
  • Must include one of: [[ ]], << >>, !commit, /commit.
  • Examples: \[\[commit.*\]\], !commit\s.
  • Recipients via @ (opt-in per notation): include @ somewhere in the notation pattern itself and any @name in the captured message becomes a recipient — both tab-completed Slack mentions (the blue @priya pill) and plain text like @sree. With notation \[\[commit.*@.*\]\], a message like [[commit @sree I'll send the report]] logs the commitment with sree as the recipient. If the notation has no @ (e.g. \[\[commit.*\]\]), captures from it ignore @-mentions in the message body. Emails (name@host.com) are always ignored.
